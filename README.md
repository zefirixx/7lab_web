## Описание задания

## Как запустить проект

1. Клонировать репозиторий:
   ```bash
   git clone <ссылка на репозиторий>
   cd lab7
Запустить контейнеры:
```bash
docker-compose up -d --build
```
Открыть в браузере:
```http://localhost:8080```

Содержимое проекта:

```docker-compose.yml``` — файл конфигурации Docker Compose для запуска контейнеров PHP и RabbitMQ, обеспечивающих асинхронную обработку сообщений через очередь

```Dockerfile``` — файл сборки контейнера PHP с установленным Composer и библиотекой php-amqplib для работы с RabbitMQ

```composer.json``` — файл конфигурации Composer с зависимостью библиотеки php-amqplib для взаимодействия с RabbitMQ

```www/index.php``` — основная страница приложения с HTML-формой для отправки данных регистрации на конференцию

```www/send.php``` — скрипт отправки данных регистрации в очередь RabbitMQ для последующей асинхронной обработки

```www/worker.php``` — скрипт worker, который прослушивает очередь RabbitMQ и обрабатывает сообщения, выполняя операции с базой данных

```www/QueueManager.php``` — класс для управления очередью сообщений RabbitMQ (создание соединения, отправка сообщений и их получение)

```www/db.php``` — файл подключения к базе данных MySQL через PDO для хранения данных регистраций

```www/vendor/``` — папка с установленными библиотеками Composer, включая php-amqplib

```composer.lock``` — файл фиксации версий зависимостей проекта для обеспечения одинаковой среды выполнения
